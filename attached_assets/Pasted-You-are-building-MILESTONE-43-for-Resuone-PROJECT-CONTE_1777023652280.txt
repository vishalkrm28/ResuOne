SECTION 1 — AI MODEL STRATEGY
==================================================

Use a multi-AI architecture.

Primary principle:
- deterministic calculations should be done in code
- AI should explain, classify, and summarize
- AI should not invent salary, tax, visa, or cost-of-living facts
- all AI outputs must be validated with Zod

AI providers:
1. OpenAI
2. Anthropic Claude
3. Google Gemini

Use OpenAI for:
- final relocation reasoning
- structured relocation score explanation
- user-facing relocation summary
- decision recommendations

Use Claude for:
- nuanced job description interpretation
- visa/language wording interpretation
- ambiguity handling
- long job descriptions

Use Gemini for:
- cheap bulk preprocessing
- batch extraction
- optional long-context job text processing

Important:
OpenAI, Claude, and Gemini all support structured outputs or structured JSON workflows, so use schema-validated outputs everywhere. OpenAI supports Structured Outputs with JSON Schema, Claude supports structured outputs/tool-use patterns, and Gemini supports structured outputs with JSON Schema. 
Reference implementation should be provider-agnostic.

SECTION 12 — RELOCATION PIPELINE
==================================================

Create:
lib/relocation/relocation-pipeline.ts

Function:
analyzeRelocationForJob({
  userId,
  candidateProfileId,
  jobId,
  internalJobId,
  lifestyle = "moderate",
  forceRefresh = false
})

Flow:
1. build cache key
2. if cached and not forceRefresh, return cached result
3. fetch job or internal job
4. fetch candidate preferences
5. fetch candidate profile
6. fetch visa signal and visa fit
7. fetch language signal and language fit
8. normalize salary
9. fetch salary benchmark if available
10. fetch city cost profile
11. estimate monthly gross salary
12. estimate monthly net salary
13. estimate monthly cost
14. calculate surplus
15. calculate relocation score
16. generate AI summary if enabled
17. save job_relocation_scores
18. update jobs/internal_jobs summary fields
19. save cache
20. return complete result

If salary is missing:
- still calculate visa/language/non-salary components
- mark salary_missing risk
- recommendation should usually be unknown or risky_move

If cost data missing:
- mark cost_data_low_confidence
- do not overstate relocation score
