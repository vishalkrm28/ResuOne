6) COUNTRY RULE ENGINE

Create:
lib/visa/country-rules.ts

Purpose:
Apply country-specific visa-sponsorship logic.

Start with Europe-focused rules.

UK:
- official sponsor list match = strong positive
- skilled worker wording = positive
- “must have right to work in UK” = negative

Germany:
- EU Blue Card wording = positive
- skilled tech/engineering role + salary present = medium positive
- relocation support = positive
- “must already hold work permit” = negative

Sweden:
- relocation support = positive
- tech/engineering/specialist role = medium positive
- “must have Swedish work permit” = negative
- sponsorship cannot be assumed without explicit signal

Netherlands:
- recognized sponsor wording = positive
- relocation support = positive
- highly skilled migrant wording = positive

General Europe:
- relocation support = positive
- international candidates welcome = positive
- right-to-work requirement = negative
- remote jobs get slight positive only if country restriction is not present

Return:

  "countryScore": 0,
  "countrySignals": [],
  "notes": ""
}

Keep these rules conservative.

7) VISA SCORING ENGINE

Create:
lib/visa/scoring.ts

Input:
{
  keywordSignals,
  companyProfile,
  countryRuleResult,
  recruiterInput,
  jobCountry,
  jobDescription
}

Scoring formula:

keywordScore:
- positive signal: +10 each, max +40
- relocation signal: +8 each, max +20
- strong negative signal: immediate "no" unless recruiter explicitly says sponsorship available

companyScore:
- known official sponsor: +30
- sponsorship_history_score:
  - 75+ = +25
  - 50-74 = +15
  - 25-49 = +5
  - below 25 = 0

countryScore:
- from country rule engine, max +20

recruiterInputScore:
- recruiter says visa sponsorship available = +30
- recruiter says relocation support = +15
- recruiter says no sponsorship = immediate "no"

Final mapping:
- strong negative signal => no
- score >= 75 => high
- score 50-74 => medium
- score 25-49 => low
- no evidence => unknown

Return:
{
  "sponsorshipSignal": "high|medium|low|no|unknown",
  "confidenceScore": 0,
  "positiveSignals": [],
  "negativeSignals": [],
  "evidenceSummary": ""
}

Rules:
- be conservative
- no guarantees
- explain evidence briefly
- if signal is unknown, say evidence is insufficient
