CURRENT PRODUCT PROBLEM
Existing direct job-board APIs are not globally reliable enough.
Coverage is inconsistent across countries like Sweden and many others.

NEW PRODUCT GOAL
Build Resuone into a GLOBAL JOB DISCOVERY ENGINE that:
1. finds jobs across countries, including Sweden
2. works for local jobs, global jobs, and remote jobs
3. uses search-discovery + ATS ingestion, not only narrow board APIs
4. normalizes jobs into one schema
5. deduplicates jobs from multiple sources
6. ranks jobs against the user’s CV/profile
7. caches and stores results for speed and cost control
8. stays modular so more sources can be added later

PRODUCT POSITIONING SHIFT
Resuone is NOT a job board.
Resuone is a global job discovery and matching engine.

IMPORTANT RULES
- Do not rely on one country-limited job API
- Use multiple source types:
  a) Google Jobs search discovery
  b) ATS-native sources (Greenhouse, Lever)
  c) remote-first sources later if useful
- Keep all sources behind modular adapters
- Normalize all jobs into one shared schema
- Deduplication is mandatory
- AI ranking happens only AFTER aggregation
- Store jobs and search results in Supabase
- Claude should only be used for reranking/matching, not for ordinary scraping
- Keep source adapters isolated from database logic
- Keep architecture ready for future ATS connectors like Workday and SmartRecruiters
