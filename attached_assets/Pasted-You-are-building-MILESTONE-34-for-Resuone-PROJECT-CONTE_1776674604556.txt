You are building MILESTONE 34 for Resuone.

PROJECT CONTEXT
Resuone already has or is implementing:
- Next.js App Router
- Supabase
- Anthropic Claude integration
- CV parsing
- CV vs job analysis
- recruiter ranking workflow
- open job recommendations based on user CV

Now build a candidate-side execution layer where the user can:
1. Pick a recommended job
2. Tailor their CV specifically for that job
3. Generate a job-specific cover letter
4. Save multiple tailored versions
5. Export or copy the outputs easily

PRIMARY GOAL
Turn Resuone from “analysis only” into “actionable application assistant”.

The user journey should be:
1. User uploads CV or uses saved parsed CV
2. User opens a recommended job or any pasted job description
3. User clicks:
   - Tailor CV
   - Generate Cover Letter
4. System creates:
   - tailored professional summary
   - improved skills emphasis
   - rewritten experience bullets aligned to job
   - ATS keyword enhancement suggestions
   - full cover letter
5. User reviews output
6. User can save versions and export/copy them

IMPORTANT RULES
- Never invent fake experience or fake achievements
- Never add skills the candidate clearly does not have
- Tailoring must stay truthful
- Improvement should come from reframing, emphasis, clarity, and better positioning
- All AI calls must happen server-side
- Save each generated output as a version
- Keep original parsed CV untouched
- Tailored CV data must be stored separately from original CV data
- Use structured JSON for tailored CV output
- Validate model outputs with Zod

TECH STACK
- Next.js App Router
- TypeScript
- Supabase
- Anthropic Claude

- Existing Resuone UI system

5) TAILOR CV ROUTE
Create:
app/api/application/tailor-cv/route.ts

Input:
{
  "userId": "...",
  "sourceCandidateDocumentId": "...",
  "candidateProfileId": "...",
  "externalJobId": "...", 
  "externalJobsCacheId": "...",
  "jobDescriptionId": "...",
  "jobText": "...",
  "versionName": "Tailored for Operations Manager"
}

Flow:
1. Resolve original parsed CV
2. Resolve job source
   - use external_jobs_cache if selected recommended job
   - otherwise use jobDescription text/pasted job
3. Build tailor prompt
4. Call Claude
5. Validate tailored CV JSON
6. Save tailored_cvs row
7. Return saved tailored CV

IMPORTANT
If job exists in external_jobs_cache, use normalized job fields plus job description.
If job is pasted manually, use provided jobText.
