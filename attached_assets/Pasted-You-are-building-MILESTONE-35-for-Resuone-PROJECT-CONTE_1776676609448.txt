1. User receives recommended jobs or finds jobs manually
2. User saves jobs they like
3. User creates tailored CV / cover letter for a saved job
4. User marks when they apply
5. User tracks application stage
6. User generates interview prep for that exact job
7. User stores notes, answers, and reminders
8. User sees all applications in one structured dashboard

IMPORTANT RULES
- Keep application records as first-class product entities
- Do not treat saved jobs and applied jobs as the same thing
- Save job metadata snapshot so changes in external provider do not break historical application records
- Keep user-generated notes editable
- AI-generated interview prep must be grounded in the actual job and user profile
- Never invent facts about the candidate
- All AI calls must happen server-side
- Save all outputs in Supabase
- Keep architecture modular for future notifications and email integrations
