Implement Milestone 19 now.

Goal:
Replace ParsePilot's unstable AI-generated match percentage with a deterministic, weighted ATS-style scoring engine that stays consistent across reanalysis.

Problem:
Right now the match score changes each time because AI is likely deciding or influencing the score directly. This creates inconsistency and reduces trust.

Core requirement:
The same CV + same job description input must always produce the same match score.

High-level strategy:
- AI may still be used for structured extraction
- AI must NOT decide the final numeric score
- The numeric score must be calculated in code using deterministic rules

--------------------------------------------------
1. Refactor the scoring architecture
--------------------------------------------------

Update the analysis pipeline so that:

A. AI is only used for structured extraction of:
- CV keywords / skills
- JD keywords / skills
- responsibilities
- optional seniority signals
- optional industry signals

B. The final match score is calculated entirely in code using a deterministic scoring function.

Do NOT allow the model to return:
- final percentage
- final score band
- final weighted score

AI can still return structured inputs used by the scoring engine, but not the final score.

--------------------------------------------------
2. Build a deterministic weighted scoring engine
--------------------------------------------------

Create a new scoring layer, for example in:

- lib/scoring/
  - normalize-keywords.ts
  - keyword-similarity.ts
  - calculate-match-score.ts
  - score-bands.ts
  - scoring-types.ts

The scoring engine should calculate score based on weighted components.

Recommended score model:

A. Required keyword match = 45%
- match against required skills / required terms from JD

B. Preferred keyword match = 15%
- match against preferred skills / nice-to-have terms

C. Responsibility alignment = 20%
- compare CV experience signals against JD responsibilities

D. Seniority / role fit = 10%
- compare years/level/title signals if available

E. Industry / domain alignment = 10%
- compare industry/domain terms where relevant

Total = 100%

Rules:
- all scoring must be deterministic
- no random weighting
- no model-generated percentage
- all inputs normalized before matching
