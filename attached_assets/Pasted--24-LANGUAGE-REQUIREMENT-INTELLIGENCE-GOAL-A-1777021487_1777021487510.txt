--------------------------------------------------
24) LANGUAGE REQUIREMENT INTELLIGENCE

GOAL
Allow candidates to understand whether a job requires local language skills, English-only work, or multilingual ability.

This must work for:
- external discovered jobs
- internal recruiter-posted jobs
- candidate recommendations
- pre-apply analysis
- notifications
- relocation-aware matching

IMPORTANT RULE
Do not assume language requirements unless there is evidence.
Show:
- English-only friendly
- Local language required
- Local language preferred
- Multilingual role
- Unknown

24.4) LANGUAGE KEYWORD DETECTION

Create:
lib/language/language-keyword-detector.ts

Detect language requirements from job descriptions.

English-friendly signals:
- English-speaking environment
- English is our working language
- no Swedish required
- no German required
- international team
- global team
- English fluency required
- business English

Local language required signals:
- Swedish required
- fluent Swedish
- German required
- Dutch required
- French required
- local language required
- must speak Swedish/German/Dutch/French
- native-level local language

Local language preferred signals:
- Swedish preferred
- German is a plus
- Dutch is a plus
- local language is a plus
- nice to have Swedish
- beneficial if you speak German

Multilingual signals:
- multilingual
- bilingual
- fluent in English and Swedish
- English plus German
- multiple languages required

Return:
{
  "requiredLanguages": [],
  "preferredLanguages": [],
  "positiveSignals": [],
  "negativeSignals": [],
  "languageRequirementSignal": "english_friendly|local_required|local_preferred|multilingual|unknown"
}

24.5) COUNTRY LANGUAGE RULES

Create:
lib/language/language-country-rules.ts

Use conservative country defaults.

Examples:
Sweden:
- if job text says Swedish required => local_required
- if job text says Swedish preferred => local_preferred
- if English working language or international team => english_friendly
- do not assume Swedish required without evidence

Germany:
- if German required => local_required
- if German preferred => local_preferred
- tech roles with English working language => english_friendly

Netherlands:
- English-friendly more common in tech/international companies
- Dutch required signals must be respected

France/Spain/Italy:
- local language requirement is more common, but still do not assume without text evidence

Return:
{
  "countryLanguageSignals": [],
  "countryLanguageScore": 0,
  "notes": ""
}

--------------------------------------------------
24.6) LANGUAGE SCORING

Create:
lib/language/language-scoring.ts

Rules:
- explicit local language required => local_required, confidence 85+
- explicit English working language => english_friendly, confidence 80+
- local language preferred => local_preferred, confidence 70+
- multiple required languages => multilingual, confidence 80+
- no evidence => unknown

Do not over-infer from country alone.

Return:
{
  "languageRequirementSignal": "",
  "requiredLanguages": [],
  "preferredLanguages": [],
  "confidenceScore": 0,
  "evidenceSummary": ""
}

24.8) LANGUAGE FIT ENGINE

Create:
lib/language/language-fit.ts

Function:
calculateCandidateLanguageFit({
  candidateKnownLanguages,
  candidatePreferredWorkingLanguages,
  requiredLanguages,
  preferredLanguages,
  languageRequirementSignal
})

Return:
{
  "languageFit": "good|risky|poor|unknown",
  "reasoning": ""
}

Rules:
- if required language is in candidate known languages => good
- if required language missing => poor
- if local language preferred but missing => risky
- if english_friendly and candidate knows English => good
- if unknown => unknown
