Now build the next practical execution layer so Resuone helps the user after they apply and before the interview.

PRIMARY GOAL
Create a workflow where the user can:
1. generate follow-up emails for applications
2. generate thank-you emails after interviews
3. schedule and track interview events
4. prepare for interviews with mock questions
5. answer mock questions in-app
6. get AI feedback on their answers
7. keep all communication and prep linked to the application

USER JOURNEY
1. User opens an application in Resuone
2. User sees actions like:
   - Generate follow-up email
   - Generate thank-you email
   - Schedule interview
   - Start mock interview
3. User saves an email draft or copies it
4. User adds interview date/time
5. User runs a mock interview session for that job
6. User writes answers to generated questions
7. Resuone evaluates the answers and gives feedback
8. User improves answers before the real interview

IMPORTANT RULES
- Never invent facts about the user
- Never fabricate achievements
- Email drafts must be grounded in the application stage and actual job
- Mock interview questions must be grounded in the job and candidate profile
- Feedback should be practical, professional, and honest
- All AI calls must happen server-side
- Save all drafts, sessions, answers, and feedback in Supabase
- Keep email generation separate from actual sending
- Keep calendar/interview scheduling internal to the product first
- Build architecture so Google Calendar / Outlook can be added later
